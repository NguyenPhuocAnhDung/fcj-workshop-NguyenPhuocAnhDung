# Summary Report: “GenAI-powered App-DB Modernization Workshop”

### Event Purpose

- Share best practices in modern application design
- Introduce Domain-Driven Design (DDD) and Event-Driven Architecture
- Guide selection of optimal compute services
- Introduce AI tools supporting the development lifecycle

- **Role:** Participant

- **Jignesh Shah** - Director, Open Source Databases

### Key Highlights

#### Drawbacks of Legacy Application Architectures

- Slow release cycles → Revenue loss / missed opportunities
- Operational inefficiency → Decreased productivity and high cost
- Non-compliance with security standards → Security risks and reputation damage

#### Transitioning to Modern Architecture - Microservices

Transitioning into a modular system where each capability is an **independent service** communicating asynchronously via **events**, built on 3 core pillars:

- **Queue Management**: Asynchronous task processing
- **Caching Strategy:** Performance optimization
- **Message Handling:** Flexible inter-service communication

#### Domain-Driven Design (DDD)

- **4-step approach**: Identify domain events → Arrange timeline → Identify actors → Define bounded contexts
- **Bookstore Case Study**: Practical DDD application example
- **Context Mapping**: 7 integration patterns for bounded contexts

#### Event-Driven Architecture

- **3 integration patterns**: Publish/Subscribe, Point-to-Point, Streaming
- **Benefits**: Loose coupling, scalability, resilience
- **Sync vs Async Comparison**: Trade-off analysis

#### Compute Evolution

- **Shared Responsibility Model**: EC2 → ECS → Fargate → Lambda
- **Serverless Benefits**: No server management, auto-scaling, pay-for-value
- **Functions vs Containers**: Selection criteria

#### Amazon Q Developer

- **SDLC Automation**: From planning to maintenance
- **Code Transformation**: Java upgrade, .NET modernization
- **AWS Transform Agents**: VMware, Mainframe, .NET migration

### Key Takeaways

#### Design Mindset

- **Business-First Approach**: Always start with the business domain, not just tech stack
- **Ubiquitous Language**: Importance of shared vocabulary between business and tech teams
- **Bounded Contexts**: Identifying and managing complexity in large systems

#### Technical Architecture

- **Event Storming Technique**: Practical method to model business processes
- Using **event-driven communication** instead of synchronous calls
- **Integration Patterns**: Understanding when to use sync, async, pub/sub, or streaming
- **Compute Spectrum**: Criteria to choose between VMs, containers, and serverless

#### Modernization Strategy

- **Phased Approach**: Clear roadmap instead of rushed migrations
- **7Rs Framework**: Multiple paths tailored to application characteristics
- **ROI Measurement**: Cost reduction + business agility

### Work Application

- **Apply DDD** to project: Event storming sessions with the team
- **Refactor Microservices**: Using bounded contexts to define service boundaries
- **Implement Event-Driven Patterns**: Replace synchronous calls with async messaging
- **Serverless Adoption**: Pilot AWS Lambda for suitable use cases
- **Explore Amazon Q Developer**: Boost development workflow productivity
